# Runner Self-Install Design

## Goal

Operators install the emerge runner on their machine with a single command — no SSH access from the CC side, no manual Python setup, no config files to edit. The daemon auto-generates the install URL with all parameters baked in.

## Architecture

```
CC developer machine                    Operator machine (any OS)
────────────────────────────────        ──────────────────────────────────
emerge daemon (localhost:8789)          curl / irm
  └── GET /runner-install.sh   ──────►  install.sh runs:
        embeds:                           1. detect OS
        - TEAM_LEAD_URL (LAN IP)          2. install Python if missing
        - PROFILE                         3. pip install deps (CN mirror?)
        - PORT                            4. download runner.tar.gz from daemon
  └── GET /runner-dist/         ──────►  5. configure autostart (platform)
        runner.tar.gz                     6. start watchdog
          (scripts/*.py bundle)           7. health check → daemon sees runner online
```

The runner install URL is shown in the cockpit and can be generated by CC via `runner-install-url` CLI command.

## Components

### 1. Daemon HTTP endpoints (scripts/daemon_http.py)

**`GET /runner-install.sh?profile=<p>`** — dynamically generated bash script for Linux/macOS.

**`GET /runner-install.ps1?profile=<p>`** — dynamically generated PowerShell script for Windows.

Both scripts embed:
- `TEAM_LEAD_URL` — daemon's LAN IP + port, auto-detected at request time
- `PROFILE` — from query param
- `RUNNER_PORT` — from profile config (default 8787)

**`GET /runner-dist/runner.tar.gz`** — tarball of runner files needed on operator machine:
- `scripts/remote_runner.py`
- `scripts/runner_watchdog.py`
- `scripts/exec_session.py`
- `scripts/runner_client.py`
- `scripts/policy_config.py`
- `requirements-runner.txt` (new file listing pip deps)

The tarball is built on-demand from local files. No caching — always reflects current code.

### 2. LAN IP detection (scripts/admin/shared.py)

New function `_detect_lan_ip() -> str`:
- Uses `socket.getfqdn()` + `socket.getaddrinfo()` to find the non-loopback IPv4 address
- Falls back to `socket.gethostbyname(socket.gethostname())`
- Never returns `127.0.0.1` or `::1`
- Cached per daemon process (IP doesn't change while daemon runs)

### 3. China detection in install scripts

Both `install.sh` and `install.ps1` run a quick connectivity probe:
```bash
# bash
if ! curl -s --max-time 3 https://pypi.org > /dev/null 2>&1; then
    USE_CN_MIRROR=1
fi
```

If CN mirror:
- pip: `--index-url https://pypi.tuna.tsinghua.edu.cn/simple`
- Python (Linux): use official mirrors at `https://registry.npmmirror.com/binary.html`
- Python (Windows): official python.org (usually accessible from China)

### 4. Python installation (per platform)

**Linux (bash)**:
- Check `python3 --version` ≥ 3.9
- If missing: try `apt-get install python3` → `yum install python3` → `dnf install python3`
- Last resort: `pyenv install 3.11` with CN mirror

**macOS (bash)**:
- Check `python3 --version` ≥ 3.9
- If missing: try `brew install python3` (if brew exists), else download pkg from python.org

**Windows (PowerShell)**:
- Check `python --version` ≥ 3.9
- If missing: try `winget install Python.Python.3.11`
- Fallback: download installer exe from python.org, run silently `/quiet InstallAllUsers=0 PrependPath=1`

### 5. Autostart configuration (per platform)

**Linux**: write `~/.config/systemd/user/emerge-runner.service`, run `systemctl --user enable --now emerge-runner`

```ini
[Unit]
Description=Emerge Runner
After=network.target

[Service]
ExecStart=/usr/bin/python3 %h/.emerge/runner/scripts/runner_watchdog.py --host 0.0.0.0 --port RUNNER_PORT
Restart=always
Environment=EMERGE_TEAM_LEAD_URL=TEAM_LEAD_URL

[Install]
WantedBy=default.target
```

**macOS**: write `~/Library/LaunchAgents/com.emerge.runner.plist`, run `launchctl load`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" ...>
<plist version="1.0"><dict>
  <key>Label</key><string>com.emerge.runner</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/bin/python3</string>
    <string>/Users/USER/.emerge/runner/scripts/runner_watchdog.py</string>
    <string>--host</string><string>0.0.0.0</string>
    <string>--port</string><string>RUNNER_PORT</string>
  </array>
  <key>EnvironmentVariables</key>
  <dict><key>EMERGE_TEAM_LEAD_URL</key><string>TEAM_LEAD_URL</string></dict>
  <key>RunAtLoad</key><true/>
  <key>KeepAlive</key><true/>
</dict></plist>
```

**Windows**: write `%USERPROFILE%\.emerge\start_emerge_runner.vbs`, add registry key:
```
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
  EmergeRunner = wscript.exe "C:\Users\<user>\.emerge\start_emerge_runner.vbs"
```

VBS content:
```vbs
Set sh = CreateObject("WScript.Shell")
sh.CurrentDirectory = "<runner_root>"
sh.Run """<python>"" ""<runner_root>\scripts\runner_watchdog.py"" --host 0.0.0.0 --port <port>", 0, False
```

### 6. Runner config file (~/.emerge/runner-config.json)

Written on the operator machine after successful install:
```json
{
  "team_lead_url": "http://192.168.1.100:8789",
  "profile": "mycader-1",
  "port": 8787,
  "installed_at": "2026-04-15T10:00:00Z"
}
```

`runner_watchdog.py` reads this on startup to pass `EMERGE_TEAM_LEAD_URL` to `remote_runner.py`.

### 7. CLI: runner-install-url (scripts/repl_admin.py)

New subcommand for CC to generate and display the install URL:

```bash
python3 scripts/repl_admin.py runner-install-url --profile mycader-1
# Output:
# Linux/macOS: curl "http://192.168.1.100:8789/runner-install.sh?profile=mycader-1" | bash
# Windows:     irm  "http://192.168.1.100:8789/runner-install.ps1?profile=mycader-1" | iex
```

Also exposed as `GET /api/control-plane/runner-install-url?profile=<p>` in cockpit API.

### 8. requirements-runner.txt (new file)

The runner uses only stdlib. Optional deps for tray icon support:
```
# optional — tray icon (skipped gracefully if missing)
pystray>=0.19; sys_platform != "linux"
Pillow>=9.0; sys_platform != "linux"
```

The install script runs `pip install -r requirements-runner.txt` with `--ignore-requires-python` so missing optional deps don't block install. Hard requirement: Python ≥ 3.9 (stdlib only).

## Install flow end-to-end

```
1. CC developer: `repl_admin.py runner-install-url --profile mycader-1`
   → prints: curl "http://192.168.1.100:8789/runner-install.sh?profile=mycader-1" | bash

2. Developer sends URL to operator (chat, email, etc.)

3. Operator runs the command on their machine.

4. install.sh:
   a. Detect OS
   b. Probe pypi.org → decide CN mirror
   c. Check Python ≥ 3.9, install if missing
   d. pip install requests (+ other deps from requirements-runner.txt)
   e. curl daemon/runner-dist/runner.tar.gz → extract to ~/.emerge/runner/
   f. Write ~/.emerge/runner-config.json
   g. Configure autostart (systemd / launchd / registry)
   h. Start watchdog now
   i. Health check: curl http://localhost:8787/health → OK
   j. Register with daemon: runner POSTs to TEAM_LEAD_URL/runner/online

5. CC sees runner online in runner-status / cockpit Monitors tab.
```

### 9. Cockpit "Add Runner" panel (cockpit_shell.html)

The Monitors tab gains an "Add Runner" section. When no runners are online, it shows prominently; when runners exist, it collapses to a small "＋ Add another runner" link.

Content (style matches existing cockpit dark theme):

```
┌─ Add Runner ──────────────────────────────────────────────┐
│  Profile:  [mycader-1      ▼]  (dropdown of known profiles│
│            + "new profile…" option)                        │
│                                                            │
│  Linux / macOS                                   [Copy]    │
│  curl "http://192.168.1.x:8789/runner-install.sh?…" | bash│
│                                                            │
│  Windows PowerShell                              [Copy]    │
│  irm  "http://192.168.1.x:8789/runner-install.ps1?…" | iex│
│                                                            │
│  Runner will appear here once it connects.                 │
└────────────────────────────────────────────────────────────┘
```

- Profile dropdown populated from `GET /api/control-plane/runner-install-url?profile=<p>`
- URLs refresh when profile changes (JS fetch)
- Copy button uses `navigator.clipboard.writeText`
- Panel polls `GET /api/control-plane/monitors` every 3s; when a new runner appears, shows a green "✓ mycader-1 connected" badge and stops polling
